# /change - Create Embedded System Change Proposal

Create a comprehensive change proposal for embedded system modifications in `specs/changes/<change-id>/`.

## Available Spec Types

- **hardware/** - Hardware component changes
- **firmware/** - Firmware module changes
- **protocols/** - Protocol changes
- **power-management/** - Power management changes

## Change Proposal Structure

Each change should include:
- Affected components and interfaces
- Hardware/firmware modifications
- Protocol updates
- Power impact analysis
- Testing requirements
- Safety considerations
